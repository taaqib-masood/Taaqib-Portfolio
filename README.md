## 🎨 Portfolio UI/UX Architecture — 9 Phase Breakdown

> This portfolio is not a template. It is a custom-engineered, brutally minimalist web application built to demonstrate frontend excellence, motion physics, and interactive AI — designed and developed in nine distinct engineering phases.

---

### Phase 1 — Swiss Brutalist Design System

The entire visual language is built on Swiss Brutalism: a design philosophy that treats structure, grid, and negative space as the aesthetic itself.

**Design tokens:**
| Token | Value | Purpose |
|---|---|---|
| `--border` | `#1a1c1c` | 1px solid borders on every element |
| `--border-radius` | `0px !important` | Zero radius, globally overridden |
| `--primary` | `#0040e0` | Accent blue — used for hover states only |
| `--accent` | `#2e5bff` | Secondary highlight |
| `--destructive` | `#ba1a1a` | Warning / critical state |
| `--background` | `#f9f9f9` | Off-white base (not pure white — intentional) |
| `--foreground` | `#1a1c1c` | Near-black, not true black |

**Typography:** `Outfit` for headings (custom weight axis), `Inter` for body. No browser defaults. Loaded via Next.js Google Fonts with zero layout shift.

**Grid:** Strict 12-column CSS Grid on desktop with `max-w-[1440px]` constraint. All sections align to this grid without exception.

---

### Phase 2 — Next.js App Architecture & Performance

**Framework:** Next.js 14 with App Router — React Server Components by default, client hydration only where motion and interactivity require it (`"use client"` directive scoped to leaf components).

**Performance:**
- `next/image` with `fill` layout and `object-cover` for zero layout shift on images
- Google Fonts served via `next/font` — no render-blocking font requests
- Framer Motion dynamically imported where possible
- Zero global CSS specificity conflicts — all design tokens via Tailwind CSS custom variables

**File Architecture:**
```
src/
├── app/
│   ├── layout.tsx       # Root layout — fonts, metadata, global providers
│   ├── page.tsx         # Single-page orchestrator — all section composition
│   └── globals.css      # Design tokens + Google Translate overrides
├── components/          # 20+ isolated, reusable UI components
├── data/
│   ├── projects.ts      # All project data — single source of truth
│   └── resume.ts        # About paragraphs, skills, experience
└── lib/utils.ts         # cn() utility — Tailwind class merging
```

---

### Phase 3 — Framer Motion Foundation

Every interactive element in this portfolio uses a single, consistent easing curve:

```ts
ease: [0.83, 0, 0.17, 1]
```

This is a custom cubic bezier that mimics mechanical, physical motion — a hard snap in, a precise deceleration out. Nothing in this UI floats or feels digital-soft. Everything moves like machined metal.

**Base animation pattern used everywhere:**
```tsx
<motion.div
  initial={{ opacity: 0, y: 40 }}
  whileInView={{ opacity: 1, y: 0 }}
  viewport={{ once: true, margin: "-50px" }}
  transition={{ duration: 0.5, ease: [0.83, 0, 0.17, 1] }}
/>
```

`viewport={{ once: true }}` ensures animations trigger exactly once — the first time the element enters the viewport. `margin: "-50px"` fires the animation just before the element is fully in view, keeping the experience fluid rather than trailing behind scroll.

---

### Phase 4 — Scroll-Driven Cinematics

**The Hero Photo "Window Wipe"**

The most visually striking animation. The hero photo is hidden on page load and uses Framer Motion's `useScroll` + `useTransform` hooks to reveal itself cinematically as the user scrolls.

```tsx
const { scrollYProgress } = useScroll({
  target: containerRef,
  offset: ["start start", "end start"]
});

// Reveals instantly within the first 15% of scroll
const clipPath = useTransform(
  scrollYProgress,
  [0, 0.15],
  ["inset(100% 0% 0% 0%)", "inset(0% 0% 0% 0%)"]
);

// Simultaneous scale pullback — zoom out from 120% to 100%
const scale = useTransform(scrollYProgress, [0, 0.15], [1.2, 1]);
```

The combined effect: the photo wipes upward like a cinema curtain lifting, while simultaneously pulling back from a slight zoom. The animation completes within the first small scroll — keeping it responsive and cinematic without forcing the user to scroll halfway down the page.

**Parallax Numbers**

Key statistics (`ParallaxNumber` component) use individual `useScroll` hooks per element, creating layered depth as the page scrolls — each number drifts at a slightly different rate creating a 3D-depth illusion.

---

### Phase 5 — Mechanical Interaction Polish

**Skill Boxes — Black Fill on Hover**

Each skill tag uses a mechanical fill transition:
```tsx
whileHover={{ backgroundColor: "#1a1c1c", color: "#f9f9f9" }}
transition={{ duration: 0.2, ease: [0.83, 0, 0.17, 1] }}
```
The hover fills instantly to black, like pressing a mechanical keyboard key. The reverse snap on mouse-leave is equally instantaneous — no lingering.

**Project Cards — Garage Door Reveal**

On hover, the project overlay (`bg-[#1d4ed8]`) slides *up* like a garage door opening, revealing the project image underneath. The text stays pinned to the top-left corner, floating above the image.

```tsx
// Overlay slides UP on hover
<motion.div
  initial={{ y: 0 }}
  whileHover={{ y: "-100%" }}
  transition={{ duration: 0.5, ease: [0.83, 0, 0.17, 1] }}
/>
```

On mouse-leave, the door slams back down with the same mechanical easing — instantly, with no softening.

**GitHub Contribution Graph**

GitHub contribution colors are mapped to authentic GitHub green palette values. The squares pulse with a subtle `opacity` animation on hover. Fully SVG-rendered with React.

### Phase 6 — Advanced Swiss Minimalist Interaction Polish

**CLI-Style Agent Stream**

The Agent terminal uses a block-cursor simulation:

```tsx
// Cursor blinks at 500ms intervals
<motion.span
  animate={{ opacity: [1, 0, 1] }}
  transition={{ duration: 0.5, repeat: Infinity, ease: "steps(1)" }}
  className="inline-block w-2 h-4 bg-[#00ff41] ml-1"
/>
```

Text renders in chunked batches rather than smooth character-by-character — giving a raw, unpolished terminal feel that matches the brutalist aesthetic.

**LTTS Project Card — Special Hover State**

The LTTS Proctoring card has a unique hover behaviour: the entire card background transitions to deep blue (`#1d4ed8`) and the project image becomes visible, creating a blue ambient glow effect. Purely CSS-transition based for 60fps performance.

**Command Palette (⌘K)**

A fully keyboard-navigable command menu using Base UI primitives. Fuzzy-search through all sections, projects, and actions. Press `⌘K` anywhere on the site to activate. Navigates to sections, opens CV download, opens GitHub.

### Phase 7 — Scroll-Driven Swiss Minimalist Animations

**About Section — Overhead Projector Slide**

The About section uses an `useScroll` hook to drive a `y` transform that slides the section content over the Hero as you scroll past it:

```tsx
const y = useTransform(scrollYProgress, [0, 1], ["100%", "0%"]);
```

This creates the illusion of a physical slide being projected over the hero section — the about content glides down from above and settles precisely into position.

**Section Entry Animations**

Every section (`Skills`, `Experience`, `Projects`, `Contact`) uses staggered `whileInView` animations with incremental `delay` values per child element:

```tsx
transition={{ delay: index * 0.05, ease: [0.83, 0, 0.17, 1] }}
```

This creates a cascade effect as lists of skills, project cards, and experience entries roll into view sequentially — never all at once.

**Scroll Progress Hairline**

A 1px horizontal line at the very top of the viewport tracks exact scroll progress across the page. Implemented as a `motion.div` with `scaleX` driven by a `useScroll` hook — grows from `0` to `1` as the user scrolls from top to bottom.

```tsx
<motion.div
  className="fixed top-0 left-0 h-[1px] bg-foreground origin-left z-[9999]"
  style={{ scaleX: scrollYProgress }}
/>
```

### Phase 9 — The Autonomous Terminal Agent

> The crown jewel of the portfolio. Rather than listing skills statically, I built a live interactive agent you can talk to.

**Visual Design — Phosphor Green Terminal**
```
Background:  #050505 (near-total black)
Text color:  #00ff41 (classic phosphor green)
Font:        monospace
Cursor:      2px × 16px blinking block
Border:      1px solid #1a1c1c
```

**How it works:**

1. **User types a message** in the input field (or clicks a suggested prompt chip)
2. **Message is sent** to `/api/chat/stream` — a Next.js Route Handler
3. **The API streams a response** token by token from the LLM
4. **Text renders in the terminal** character by character — the cursor blinks while streaming
5. **The agent understands context** — it's prompted to respond as Taaqib's personal AI with detailed knowledge of every project, tech stack, experience, and skill

**Pre-programmed context the agent knows:**
- Full project details for all 6 portfolio projects (LTTS, MCP, Reva AI, Stock Forecasting, Predictive Maintenance, Salon SaaS)
- Complete skills matrix across AI/ML, Full-Stack, DevOps
- Work experience at L&T Technology Services
- Location (Dubai), availability, and contact information
- Can answer recruiter-specific questions like "Is he available?" or "What's his strongest skill?"

**Mechanical Agent Teaser (McpTeaser)**

Above the main agent terminal, an SVG-rendered brutalist node map visualises the multi-agent architecture. Pure CSS/SVG — no canvas, no D3. Each node is a `+` symbol on a black grid representing a conceptual "tool" the agent has access to. On hover, nodes pulse with a mechanical highlight.

**Cross-Section Agent Triggers**

Clicking "Ask Agent" on a project card does three things simultaneously:
1. Pre-fills the agent input with `"Tell me about the [Project Name] project."`
2. Smooth-scrolls to the terminal section
3. Fires the `wakeUpAgent` custom event — picked up by a `useEffect` listener in `page.tsx`

This deep-link behaviour turns the agent from a passive chat widget into an active, integrated part of the portfolio narrative.
